

1. 数据区域（企业级数据架构)：
src：贴源层数据( source data area )，存放上游源系统的临时数据，每天日终跑批时会全部清除，有的系统按照周一到周日建7个数据库，每天清空和使用对应的数据库。
ods：源系统数据( original data system ),存放源系统数据，保存历史，不做数据标准化，不做跨系统数据整合。
dws：数据仓库( data warehous system ),  按主题存储数据，保存历史，标准化，数据质量，元数据管理，数据整合。面向企业级业务建模，网状数据模型。
dms：数据集市( data market system )，数据来自数据仓库，星型或雪花型的多维数据。 企业级多维度数据，或部门级多维度数据，例如银行的五大数据集市：零售、对公、运营、风险、财务等。
app：应用数据( application system )，面向业务应用建模。


2. 数据流向：
src => ods               => 外部客户  
src => ods        => app => 外部客户  
src => dws => dms        => 外部客户
src => dws => dms => app => 外部客户

ods、dms、app：可以直接向外部提供数据
